Plugins

Specify 6 interfaces with Google Earth and the BioGeomancer and GEOLocate web services. Choose System > Plugins.

These web services can be launched by

  • Clicking the relevant icon in the Sidebar and choosing a Catalog Number or Record Set.

  • Dragging and dropping a Collection Object, Locality or Collecting Event Record Set on the web service icon from within the sidebar. These Records Sets must include the Locality data listed below for each service.

Plugins that can accept a Record Set display a colored outline as the Record Set is dragged within the sidebar. An arrow will display within the outline when the Record Set can be dropped onto the plugin.

Specify is not responsible for the integrity or accuracy of any of the results of these web services. Specify is integrated with these services as a service to the community. For more information about each visit them at their web site by clicking them here: BioGeomancer, GEOLocate, Google Earth.

 

Google Earth

The Google Earth Plugin is used as a means of visualizing object records from Specify in Google Earth.

Google Earth must be installed on the local machine for the Specify Google Earth integration to work.

Specify plots locality records (or associated Collection Objects or Collecting Events) in Google Earth based on latitude and longitude data provided in the locality fields. All associated data is stored in a KML file which is launched within Google Earth upon clicking the Google Earth icon on the sidebar, or dropping a Record Set on Google Earth from within the Plugins Sidebar.

Collection Object information is shown within the Google Earth information balloon. Specify also allows the insertion of two web service links for genus and species within the information balloon by configuring Google Earth with URLs to the web services in Specify > Preferences > Tools.

Google Earth can also be launched from within a Locality form by clicking the Display in Google Earth icon.

Locality data Requirements

  • Latitude and Longitude

Google Earth is also available for use in the WorkBench

 

BioGeomancer and GEOLocate

BioGeomancer and GEOLocate both use a description of a Locality, and at least three geographic locations to find Latitude and Longitude data. This is referred to as georeferencing.

Specify currently interfaces with BioGeomancer classic.

Locality data Requirements

  • Geographic Ranks; including county, state and county for US data.
  • Locality Name

One weblink button is available on the Locality form for a quick link to one of these services. Choose Specify > Preferences > System for the Mac or Edit > Preferences > System for Windows and Linux to select a service.

Once the required information is entered in the Locality form, the button (at the bottom of the locality form) becomes enabled. Click the button to launch the service. The georeferencing data displays in a dialog which includes the locality information, count of results, longitude, latitude, and a map of the location. If the web service results represent more than one record each record can be stepped through by using the buttons at the bottom of the dialog:

Button Result
Help Launches Specify Help.
Accept

Accepts the locality data from the web service and copies it into the latitude and longitude fields. Specify also writes the name of the web service in the src lat long unit field, then steps to the next record.
Skip Skips the current record information and steps to the next record.
Quit Closes the dialog.

If the web service data is accepted and latitude and longitude information already exists in the locality form the existing information, including the source field, will be overwritten by the web service data.

Note: The results dialog for BioGeomancer displays more information than latitude and longitude but Specify does not keep any other data that is returned by these web services.
