LSID

LSID Overview

A field for a unique identifier exists within the Collection Object, Taxon, Geography, Lithostratigraphy, Locality, Agent, Reference Work and Journal tables in Specify. The field is named Globally Unique Identifier, (GUID).

A more specific form of GUID, Life Science Identifiers (LSIDs) have become the universally accepted identification scheme for the biological collections community. Using an LSID to identify information allows that information to be shared and recognized globally. As more biological information systems use LSIDs it will become possible to instantly recognize when globally exchanged information has been updated or duplicated.

Specify has adapted its own scheme for creating LSIDs, but was also written to include the flexibility for institutions wanting to include their own scheme to create a plugin.

Specify’s LSID scheme was created with specific sections so that information about the referenced information is instantly recognizable. This scheme includes a version number, which will change each time the record is updated.

 

LSID Authority refers to the source of the data, which is typically the url for the institution. This is edited using the Institution form (System > System Setup > Configuration).
Institution Code is the code located within Specify for the Institution. This is edited using the Institution form (System > System Setup > Configuration).
Collection Code is the code for the collection. This is edited using the Collection form (System > System Setup > Configuration).
Collecton Code refers to the type of record. This is edited using the Collection form (System > System Setup > Configuration)
Catalog Number is the catalog number for the data in Specify.
Version refers to the amount of times the information has been updated.

The Catalog Number is used within the LSID schema to identify specimen records, but all other records are identified using the primary record id found within MySQL.

LSIDs can be automatically created by Specify for each specimen, taxonomy, geography, lithostratigraphy, locality, person, reference work and journal record located within Specify. LSIDs are configured in the System Preferences.

 

LSID Preference

For Mac: Choose Specify > Preferences > LSID.
For Windows and Linux: Choose Edit > Preferences > LSID.

Note: The LSID preference is only available to the Administrator by default in Specify. If LSID does not appear in your system preferences tool bar please visit your Specify Administrator to have your permissions changed.

LSID Preference

  • Automatic LSID Generation
    Check the box to choose tables for automatically create LSIDs. This will allow new records in each specific table to include an automatically generated LSID. To automatically generate LSIDs for existing records in tables first choose the tables in this section, then click the Create LSIDs button.
  • Additional Options
    Click the Create LSIDs button to automatically generate LSIDs for existing records in the tables selected in the Automatic LSID Generation section.

Specify will not override LSID information.
